## Implemented architecture and safety changes

- Centralized agent command policy, MCP validation, exact tool lists, and environment allowlisting.
- Added a worker-workspace preparation/cleanup boundary.
- Made CAPTCHA and email authority explicit in prompt generation.
- Added marked, blank browser-profile lifecycle rules.
- Added redirect-safe manual fetching and transaction-ownership semantics.
- Added process ownership and installer deletion guards.
- Added durable `SPEC.md` goals, interfaces, invariants, tasks, and bug backpropagation history.

## Roadmap

### P0 — complete

- C-01 through C-04.
- H-01 through H-04.
- M-01.
- Full tests/lint/audit/build gates.

### P1 — security boundary

1. Put Playwright MCP/browser in a container or dedicated restricted OS account with only one worker directory mounted.
2. Add an outbound policy proxy: block private/reserved networks after resolution, pin DNS answers, and require an approved application-origin chain.
3. Require user confirmation before a cross-origin jump or final submission when the destination is outside a known ATS allowlist.

### P2 — architecture and QA

1. Extract `AgentPolicy`, `WorkerWorkspace`, `AgentRunner`, `ApplyResultParser`, and `QueueCoordinator` from `apply/launcher.py`.
2. Replace loose job/result dictionaries with typed dataclasses or validated models.
3. Add mutation tests for security invariants; raise branch floor as coverage grows.
4. Generate an SBOM/provenance attestation in publish CI.

### P3 — performance and operations

1. Instrument stage latency, retry time, browser startup, prompt size, and per-job token/cost metrics.
2. Bound LLM retry wall-clock time; use provider-specific timeout profiles.
3. Prewarm or pool safe immutable resources while retaining per-job worker/process isolation.
4. Consolidate dashboard/analytics queries where profiling shows repeated scans; preserve existing WAL/busy-timeout configuration.

## Recommendations by discipline

### Architecture

- Pull policy and lifecycle decisions down into small, typed modules; keep the CLI as composition only.
- Treat the browser agent as a hostile-input interpreter. Prompts are defense in depth, not authorization.
- Make every persisted/transient artifact owner explicit: user, job, worker, or process.

### Security

- Use client sandboxing plus OS/container isolation; [Playwright MCP states that it is not a security boundary](https://github.com/microsoft/playwright-mcp#security).
- Keep Codex/Claude least-privilege controls aligned with current [OpenAI Codex configuration](https://developers.openai.com/codex/config-reference) and [Claude Code permission modes](https://code.claude.com/docs/en/permission-modes).
- Add network-layer enforcement for DNS rebinding/private-network access and cross-origin exfiltration.
- Pin GitHub Actions to full commit SHAs following [GitHub's action hardening guidance](https://docs.github.com/en/actions/security-for-github-actions/security-guides/security-hardening-for-github-actions#using-third-party-actions).

### Performance

- Optimize measured end-to-end job latency, not individual helpers.
- The prompt simplification already reduces context/tool confusion; track prompt bytes/tokens as a regression metric.
- Cap retry wall-clock budgets and distinguish connect, read, provider throttling, and model-generation latency.

### Technical debt

- Make Docker consume a pinned base digest and locked dependency export.
- Break up high-complexity orchestration before adding more providers or browser capabilities.
- Remove dead personal-profile discovery code after a compatibility window.
- Document and automate retention for logs, old manual prompt staging, and quarantined legacy browser profiles.
